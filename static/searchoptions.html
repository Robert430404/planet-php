Search Options

Fulltext Search

Fulltext search in content and title: Liip

We use the MySQL boolean fulltext search for that, so you can use all those parameters here, but forget about the relevancy stuff, we always order the output by date.

Examples:
Find rows that contain at least one of the two words: Liip blog
Find rows that contain both words "Liip" and "blog": +Liip +blog
Find rows that contain the word “Liip” but not “blog”: +Liip -blog
Find rows that contain the exact phrase “Liip blog”: "Liip blog"

No paid postings

Just add publireport=off to your search URL and paid postings from well-known companies don't show up anymore.

Tags

tag/

Search all posts with a certain tag
Example: /tag/bloggertreffen

Search all posts with two tags
Example: /tag/bloggertreffen+zurich

Search all posts with an exact tag
Example /tag/!politik

The last one is needed, since I introduced taggroups (that's what usually is displayed in the right column). "politik" and "politics" build the taggroup "politics", so if you search for /tag/politik or /tag/politics all posts tagged as "politics" or "politik" are displayed (with both queries). But if you search for /tag/!politik really only posts with "politik" are shown and not those with "politics".

btag:

Searches all posts from a blog with a certain tag from list.blogug.ch.

Examples:
politics, Liip, namics_alumni

URLs

site:

Search all posts from a site/blog
Example: site:blog.Liip.ch

inurl:

Search all posts with "something" in the entry link
Example: inurl:klum

Links

link:

Search all posts, which link to a site/blog
Example: link:blog.Liip.ch

linkex:

Search all posts, which link to an entry
Example: linkex:www.sequenz.ch/blog/archive/[...]bloggertreffen-2.html

Language

lang:

You can search for posts only written in a certain language (only german, english, french and italian are supported right now). It's an automatic detection, so won't be correct 100% of the time:
Examples
German: lang:de
English: lang:en
French: lang:fr
Italian: lang:it

More info about this here.

Location

Search by city or canton (defined by the geolocation information provided in blogs)

Examples
city:Zürich, city:Bern, city:Sankt Gallen
canton:Zürich, canton:Bern, canton:Sankt Gallen

More info about this here.

Blog ID

id:

Each blog listed on list.blogug.ch has an unique ID and you can search with that as well.
Example: id:24

Microformats

micro:calendar
micro:card
micro:listing
micro:review
micro:formats

You can search for microformats-annotated blog posts

Examples
micro:calendar micro:card micro:listing micro:review
micro:formats (all)

micro:podcast

Not really a microformat, but I didn't find a better category. Search for all podcasts.

Examples
micro:podcast

Top Talk

Bored with reading blog posts which noone else interests? Just want to read, what everyone else is talking about and how they do it? Then look no further than top:talk (or as Atom-Feed) :) It just shows all the recent blogposts, which link to the "Top Links last 7 days" from the right side (and the Top Linked-to post itself) and leaves out the boring rest :)

Combining searches

"and" and "or" are supported for combining search queries.
For example inlink:Liip.ch or inlink:blogug.ch shows all the blog posts, which link to something with Liip.ch or blogug.ch in their links.

Parenthesis don't work yet, btw...

If you do a fulltext search, you don't have to add "or" and "and", eg. "Liip blogug" searches for all posts containing the word "Liip" or "blogug", on the other hand "+Liip +blogug" searches for all posts, which contain both terms. "and/or" does work, too, but is much slower on the DB server side"

Feeds

All the output is also available as Atom or RSS feed, just prepend the URL with /atom/ or /rss/, eg: /atom/search/link:blog.Liip.ch
